# Безопасность токена Telegram Bot API

## ⚠️ ВАЖНО: Никогда не передавайте токен третьим лицам

### Как защитить токен:

#### ✅ Безопасные способы:

1. **Локальные переменные окружения:**

   ```bash
   export BOT_TOKEN="your_token"
   ./bin/tg-timer
   ```

2. **Локальный .env файл (только для разработки):**

   ```bash
   # .env файл в .gitignore, не попадает в репозиторий
   BOT_TOKEN=your_token
   ```

3. **Системные переменные (продакшен):**

   ```bash
   # В systemd service file
   Environment="BOT_TOKEN=your_token"

   # В Docker
   ENV BOT_TOKEN=your_token

   # В Kubernetes secret
   # Используйте Kubernetes Secrets
   ```

#### ❌ Опасные способы (НИКОГДА не делайте):

1. **Не храните токен в коде**
2. **Не коммитьте .env файлы**
3. **Не отправляйте токен в сообщениях**
4. **Не публикуйте токен в публичных местах**

### Проверка безопасности:

- [ ] `.env` добавлен в `.gitignore`
- [ ] Токен только в локальных переменных
- [ ] Нет токена в коде проекта
- [ ] Используются системные переменные в проде

### Если токен скомпрометирован:

1. Зайдите в @BotFather в Telegram
2. Выполните команду `/revoke`
3. Выберите нужного бота
4. Создайте новый токен
5. Обновите переменные окружения

### Рекомендации:

- Используйте разные токены для разработки и продакшена
- Регулярно проверяйте активность бота
- Используйте webhook вместо long polling в продакшене
- Ограничьте права бота (только сообщения, без админских прав)
